⚙ UI Elements & Visual Effects

Territory Border Glow

Map Overlay
Each empire's territory has a subtle animated border glow in their faction color. Inside the territory, a faint vignette tint shows ownership. When borders shift (conquer/lose), a pulse animation ripples along the new border line. At war: border flashes angry red. At peace: calm steady glow. Allied: dual-color blend.
Rome Red / Greece Blue / Celts Green / Persia Gold

Battle Smoke & Dust

Combat VFX
When armies clash, particle effects of dust clouds and smoke rise from battle locations on the map. Arrows show as streaking lines between units. Cavalry charges leave trail effects. Victorious battles show a brief golden pillar of light. Defeated armies show fading skull markers. Siege battles show burning city effects.

Unit Card Design

UI Component
Each unit type gets a card with: stylized portrait on left (dark vignette), stats bars in center (attack/defense/speed/morale as colored bars), faction emblem watermark behind stats, golden border with tier stars (1-4), and a small animated icon showing the unit's weapon/type. Cards have parchment texture background.
How to use these concepts:
1. Each card includes an AI image generation prompt (in the monospace box) — paste it into any image generator (Midjourney, DALL-E, Stable Diffusion, ComfyUI) to create concept art
2. The color palettes define the faction/terrain identity — use these hex values when coding the game's color scheme
3. Each unit/structure has game mechanics notes (terrain bonuses, effects) — integrate these into your combat.js and map.js logic
4. Start with the base game units (Tier 1-2), then add elite units as the player progresses
5. All structures are designed to be isometric or top-down compatible for your web game renderer
